# Elakka — Product Requirements Document

## Objective

Build a **private, offline-first farm management mobile app** for a cardamom farmer in Idukki.

## Product Identity

Elakka is a **personal agricultural record system**, not a SaaS platform.

## Core Problem

Farm data is:

* Scattered (paper, WhatsApp)
* Unstructured
* Easily lost

## Target User

* Single farmer (owner)
* Idukki (Cardamom Hill Reserve)

## Core Features

* Plot management
* Treatment logs
* Yield tracking
* Inventory tracking
* Problem tracking
* Schedule/calendar
* Weather logging
* PDF export
* Secure access (PIN/biometric)

## Non-Goals

* No AI (future feature)
* No multi-user support
* No cloud dependency
* No analytics/tracking

## Success Metrics

* Daily usability
* Reliable offline performance
* Accurate farm record tracking
